# Getting Started — QA Assistants のインストールと初期設定

QA Assistants を WordPress に導入し、最初のアシスタントを動かすまでの手順です。

## 必要な環境

| 項目 | 推奨 |
|---|---|
| WordPress | 6.0 以上 |
| PHP | 7.4 以上 |
| メモリ | 128 MB 以上 |
| ブラウザ | Chrome / Firefox / Safari の最新版 |

## インストール

### 方法 1: GitHub Releases から zip を入手

1. [最新リリース](https://github.com/quarka-org/qa-assistants/releases/latest) を開く
2. **Assets** から `qa-heatmap-analytics.zip` をダウンロード
3. WordPress 管理画面 → **プラグイン** → **新規プラグインを追加**
4. **プラグインのアップロード** → ファイルを選択 → ダウンロードした zip を選択 → **今すぐインストール**
5. **プラグインを有効化**

### 方法 2: WordPress.org 経由（安定版）

1. WordPress 管理画面 → **プラグイン** → **新規プラグインを追加**
2. 検索欄に `QA Heatmap Analytics` と入力
3. インストール → 有効化

> マニフェストランタイム機能（アシスタントプラグイン作成のための新方式）を含む最新版が必要な場合は、**方法 1（GitHub Releases）** を使ってください。

## 初期設定

1. 有効化後、左メニューに **「QA ZERO」** または **「QA Assistants」** が表示されます
2. 初回起動時にトラッキング ID（計測単位の識別子）が自動発行されます
3. **設定** → **計測対象サイト**で、計測対象の URL を確認してください

## アシスタントの利用

1. 左メニュー → **AI アシスタント**
2. アシスタント一覧（カード形式）から、使いたいアシスタントをクリック
3. 対話形式で進めると、データ取得や分析が自動で実行されます

## 動作確認

最初に試すと分かりやすいアシスタント:

| アシスタント | 内容 |
|---|---|
| **サンプルアシスタント**（同梱）| マニフェストランタイムの全機能テスト用。期間選択 → ページ集計 → 詳細表示まで一通り動きます |

データが表示されない場合:
- 計測タグがサイトに設置されているか確認（**設定** > **計測対象サイト**）
- 数日分のデータが蓄積されているか確認（**訪問レポート** に PV が出るかどうか）

## トラブルシューティング

| 症状 | 確認すること |
|---|---|
| プラグイン有効化時にエラー | PHP バージョン（7.4 以上） / メモリ上限 |
| AI アシスタント画面が真っ白 | ブラウザのコンソールにエラーがないか / 他のプラグインとの競合 |
| データが「0」のまま | 計測タグが正しく設置されているか / Cron が動作しているか |

解決しない場合は [Issue を起票](https://github.com/quarka-org/qa-assistants/issues/new/choose) してください。
